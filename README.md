# interactive-cv

**Open-source-шаблон персонального интерактивного сайта-резюме с AI-ассистентом.** Любой разработчик может развернуть такой сайт под своё CV: меняется только контент (markdown) и домен, код остаётся прежним.

Живой пример сервиса, собранного из этого шаблона: **[cv.libera.pro](https://cv.libera.pro)**.

## Что это

Обычное резюме — статичный PDF. Этот проект превращает CV в интерактивный сайт, который выгодно выделяет кандидата среди сотен стандартных откликов:

- 🤖 **AI-ассистент** — рекрутёр задаёт вопросы в чат («какие проекты на Flutter?», «делал ли DevOps?»), а модель отвечает **по фактам из CV**, с защитой от галлюцинаций и выдумывания
- 🕸 **Граф знаний** — визуализация связей проектов и технологий
- 🔗 **Короткие ссылки с TTL** — персональная ссылка под каждую вакансию с аналитикой открытий
- 📄 **SSR-страницы CV** — каждый вариант резюме рендерится на сервере, с превью для мессенджеров

## Архитектура (два репозитория)

| Репозиторий | Стек | Назначение |
|-------------|------|-----------|
| [cv-backend](https://github.com/interactive-cv/cv-backend) | Python, FastAPI, PostgreSQL | REST API, RAG-промптинг, стриминг ответов LLM, аналитика ссылок |
| [cv-frontend](https://github.com/interactive-cv/cv-frontend) | Next.js 16, TypeScript, Tailwind | SSR-лендинг, граф, чат-виджет со стримингом, OG-превью |

## Технологические решения, которые стоит увидеть

- **RAG без векторной БД** — для объёма одного CV весь текст подаётся как контекст в системном промпте (дешевле и проще pgvector)
- **Защита от галлюцинаций** — жёсткий промпт + e2e-тесты против реальной модели на провокационных запросах
- **Graceful degradation** — при сбое LLM чат не падает, а отдаёт контакты; остальной сайт работает
- **Кросс-БД типы** — тесты на in-memory SQLite, прод на PostgreSQL с нативными `jsonb`/`uuid` без изменения кода
- **Атомарный инкремент** счётчика кликов без race condition через SQL `UPDATE ... WHERE`
- **Авто-TLS** — собственный nginx-образ с Let's Encrypt, сертификаты получаются и продлеваются автоматически

## Развёртывание

Прод-окружение — собственный VPS с Docker Compose. Весь стек (nginx + FastAPI + Next.js + PostgreSQL) поднимается одной командой. Подробности — в README обоих репозиториев.

## Дорожная карта

Сделать шаблон полностью параметрическим — вынести все захардкоженные значения (домен, имя, контакты, навыки) в конфигурацию, чтобы чужое CV разворачивалось только заменой контента. Детали — в [TODO](https://github.com/interactive-cv/cv-backend/blob/master/TODO.md).
